Expand description

Configuration for a Db instance.

All fields have sensible defaults via DbConfig::default(). The configuration is validated when passed to Db::open.

§Example

use aeternusdb::DbConfig;

// Use defaults (64 KiB buffer, 2 background threads)
let config = DbConfig::default();

// Or customize
let config = DbConfig {
    write_buffer_size: 128 * 1024,
    thread_pool_size: 4,
    ..DbConfig::default()
};

Fields§

§write_buffer_size: usize

Maximum size of the in-memory write buffer in bytes.

When the buffer is full, it is frozen and flushed to an SSTable in the background.

Bounds: 1 024 ≤ write_buffer_size ≤ 268 435 456 (256 MiB).

Default: 65 536 (64 KiB).

§compaction_strategy: CompactionStrategyType

Compaction strategy family.

Determines how SSTables are grouped and merged during minor, tombstone, and major compaction.

Default: CompactionStrategyType::Stcs (Size-Tiered).

§min_compaction_threshold: usize

Minimum number of similarly-sized SSTables required to trigger background minor compaction.

Bounds: 2 ≤ min_compaction_threshold ≤ 64.

Default: 4.

§max_compaction_threshold: usize

Maximum number of SSTables to merge in a single minor compaction.

Bounds: min_compaction_threshold ≤ max_compaction_threshold ≤ 256.

Default: 32.

§tombstone_compaction_ratio: f64

Tombstone-to-total-record ratio that triggers background tombstone compaction on an SSTable.

Bounds: 0.0 < tombstone_compaction_ratio ≤ 1.0.

Default: 0.3.

§tombstone_compaction_interval: usize

Minimum SSTable age (in seconds) before it is eligible for tombstone compaction. Set to 0 to disable the age gate.

Bounds: 0 ≤ tombstone_compaction_interval ≤ 604 800 (7 days).

Default: 0 (no minimum age).

§tombstone_bloom_fallback: bool

When true, tombstone compaction resolves bloom filter false positives by performing an actual get() against older SSTables for each point tombstone. This enables more aggressive tombstone removal at the cost of extra I/O.

Default: true.

§tombstone_range_drop: bool

When true, tombstone compaction scans older SSTables to check whether a range tombstone still covers any live keys, allowing aggressive range tombstone removal.

Default: true.

§thread_pool_size: usize

Number of background worker threads for flushing and compaction.

Bounds: 1 ≤ thread_pool_size ≤ 32.

Default: 2.
