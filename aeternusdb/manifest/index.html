Â§Data durability strategy

Manifest metadata is persisted using a WAL + periodic snapshot model:

  1. Manifest WAL (manifest.wal) records mutation operations:

    • switching active WAL,
    • promoting WALs to frozen,
    • adding/removing SSTables,
    • updating LSN.

  2. Manifest snapshot (MANIFEST-000001) is a compact encoded dump of the whole metadata structure. Checksum ensures corruption detection.

  3. On startup:

    • If a valid snapshot exists â†’ load snapshot, replay WAL.
    • If snapshot corrupted â†’ return error.

This ensures crash recovery is always correct and consistent.
